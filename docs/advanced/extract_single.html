Extract from Other Response
Acquire a value from the response of another HTTP request

CPH can acquire replacement values from other HTTP responses by issuing a configured request prior to modifying the current HTTP message.

Consider, for example, a mechanism similar to some anti-CSRF tokens:

  1. Client requests page
  2. Server responds with page + unique value
  3. Client must submit its next request with that unique value

To acquire a value from the response of another HTTP request:

  1. Either...
    • Find the desired HTTP request in Burp Suite, give it a right-click, and choose Send to CPH; or
    • Select the appropriate drop-down option in step 3 within the Parameter handling pane and paste a raw HTTP request in the left pane.
  2. Use the Issue button to confirm that the configured HTTP request will yield the desired response;
  3. Configure an appropriate regular expression to extract the desired value from that response.

Subsequent requests within scope and containing targeted matches will cause CPH to automatically issue the tab's configured request in order to extract the modification value from its response.
